# Idle Hacking Market Companion

Minimal public GitHub Pages companion site for Idle Hacking market snapshots.

## Run

From the repository root, refresh local public data first:

```bash
python3 tools/update_local_market_site.py
```

This regenerates compact history and runs the operator checker as the final trust gate. Trust localhost only after this command passes.

```bash
cd site
npm install
npm run dev -- --host 127.0.0.1
```

## Build

```bash
cd site
npm run build
```

## Preview

```bash
cd site
npm run preview
```

## Notes

- The app loads public data from `public/data/latest.json`.
- The app also reads `public/data/index.json` for snapshot count and advertised ranges.
- Compact public history files live under `public/data/history/` for upcoming graph/detail work.
- Compact public history is regenerated by `python3 tools/update_local_market_site.py`.
- `tools/import_public_market_data.py --no-pull` is for deliberate offline/regression testing only.
- The market freshness/status line is part of the public UI.
- `vite.config.js` currently uses `base: './'` so the build is suitable for GitHub Pages subpaths.
- If you later deploy to a different base path, adjust `base` in `site/vite.config.js`.
- Current planning is in `src/prototype/MARKET_HISTORY_AND_GRAPHS_PLAN.md`.

## Health Check

From the repository root:

python3 tools/check_public_market_health.py
